# API使用ガイド

## 概要

zcrc-chronos APIは非同期処理ベースの時系列予測サービスを提供します。長時間実行される予測処理をバックグラウンドで実行し、進捗状況やキャンセルを管理できます。

## 利用可能なモデル

以下のモデルが利用可能です：

#### デフォルト・汎用モデル
| モデル名 | 説明 | 実行時間 | 推奨用途 |
|---------|------|----------|----------|
| `chronos_default` | **デフォルト設定**（9種類のモデルから自動選択）※詳細は下記参照 | 15分以内 | 汎用的な予測タスク |
| `fast_statistical` | 統計的手法中心（SeasonalNaive、AutoETS等） | 5分以内 | 高速処理、短期予測 |
| `balanced_ml` | 機械学習ベース（NPTS、DynamicOptimizedTheta等） | 15分以内 | バランス重視、中期予測 |
| `deep_learning` | 深層学習ベース（ChronosZeroShot、TFT、DeepAR等） | 30分以内 | 最高精度、長期予測 |

#### 単一モデル（特化型）
| モデル名 | 説明 | 実行時間 | 推奨用途 |
|---------|------|----------|----------|
| `autoets_only` | AutoETSモデルのみ（統計的手法） | 5分以内 | 短期予測、統計的手法 |
| `npts_only` | NPTSモデルのみ（Neural Prophet） | 10分以内 | 中期予測、機械学習 |
| `seasonal_naive_only` | SeasonalNaiveモデルのみ（季節性ベースライン） | 2分以内 | ベースライン予測、超高速処理 |
| `recursive_tabular_only` | RecursiveTabularモデルのみ（勾配ブースティング） | 10分以内 | 中長期予測、高精度 |
| `ets_only` | ETSモデルのみ（標準的な指数平滑法） | 5分以内 | 短期予測、統計的手法 |

#### Zero Shotモデル（事前訓練済み）
| モデル名 | 説明 | 実行時間 | 推奨用途 |
|---------|------|----------|----------|
| `chronos_bolt` | **Chronos-Bolt高速**（事前訓練済みTransformer） | 5分以内 | 訓練不要の即座予測 |


## chronos_defaultモデルの詳細

`chronos_default`は**AutoGluonの自動モデル選択機能**を使用し、入力データに最適なモデルを9種類から自動的に選択・訓練します。

### 自動選択されるモデル一覧

以下のモデルから、データの特性と予測期間に応じて最適なモデルが自動選択されます：

| # | モデル名 | 種類 | 説明 | 特徴 |
|---|---------|------|------|------|
| 1 | **SeasonalNaive** | 統計的 | 季節性ナイーブモデル | シンプル、季節パターン重視 |
| 2 | **AutoETS** | 統計的 | 自動指数平滑法 | エラー・トレンド・季節性を自動選択 |
| 3 | **NPTS** | 機械学習 | Neural Prophet Time Series | 非パラメトリック時系列学習 |
| 4 | **RecursiveTabular** | 機械学習 | 勾配ブースティング | 回帰問題として時系列予測 |
| 5 | **DirectTabular** | 機械学習 | 直接的なタブラーモデル | 表形式データとして処理 |
| 6 | **DynamicOptimizedTheta** | 統計的 | 動的最適化Thetaモデル | バランス型、中精度 |
| 7 | **DeepAR** | 深層学習 | Amazon開発RNNモデル | 確率的深層学習 |
| 8 | **PatchTST** | 深層学習 | Patch-based Transformer | 効率的なTransformer |
| 9 | **TiDE** | 深層学習 | Time-series Dense Encoder | 密結合エンコーダー |

### 除外されるモデル

以下のモデルは品質や安定性の理由で除外されています：

| モデル名 | 除外理由 |
|---------|----------|
| **Naive** | 予測品質が低い（単純な値の繰り返し） |
| **ChronosZeroShot** | 個別モデル`chronos_bolt`として提供 |
| **ChronosFineTuned** | 不安定な動作のため除外 |
| **TemporalFusionTransformer** | 計算コストが高く、個別モデル`deep_learning`で利用可能 |

### エラー時のフォールバックモデル

初回学習でエラーが発生した場合、予測期間に応じてより軽量なモデルで再試行されます：

#### 短期予測（≤6時間）
- **ETS** - 標準的な指数平滑法
- **SeasonalNaive** - 季節性ナイーブ
- **RecursiveTabular** - 勾配ブースティング

#### 中長期予測（>6時間）
- **SeasonalNaive** - 季節性ナイーブ
- **ETS** - 標準的な指数平滑法
- **Theta** - Thetaモデル
- **RecursiveTabular** - 勾配ブースティング
- **Chronos** - Chronosモデル（フォールバック時のみ利用）

### 選択アルゴリズム

AutoGluonは以下の基準でモデルを評価・選択します：

1. **データサイズ** - 利用可能なデータ量
2. **季節性パターン** - データの周期性
3. **トレンドの強さ** - 上昇・下降傾向
4. **ノイズレベル** - データの変動度
5. **予測期間** - 短期・中期・長期予測
6. **検証スコア** - MAE（平均絶対誤差）による評価

最終的に**最も高い検証スコア**を達成したモデルが選択され、予測に使用されます。

## API使用フロー

### 1. 非同期予測の開始

```bash
curl -X POST "http://localhost:8000/api/v1/predict_zero_shot_async" \
  -H "Content-Type: application/json" \
  -d '{
    "timestamp": [
      "2023-01-01T00:00:00",
      "2023-01-01T01:00:00",
      "2023-01-01T02:00:00"
    ],
    "values": [10.5, 11.2, 10.8],
    "forecast_until": "2023-01-04T02:00:00",
    "model_name": "autoets_only",
    "model_params": {}
  }'
```

**レスポンス:**
```json
{
  "task_id": "12345678-1234-5678-9012-123456789012",
  "status": "pending",
  "message": "予測タスクが正常に開始されました"
}
```

### 2. 予測ステータスの確認

```bash
curl -X GET "http://localhost:8000/api/v1/prediction_status/12345678-1234-5678-9012-123456789012"
```

**実行中のレスポンス:**
```json
{
  "task_id": "12345678-1234-5678-9012-123456789012",
  "status": "running",
  "progress": 0.6,
  "message": "モデル訓練中...",
  "created_at": "2023-01-01T00:00:00Z",
  "updated_at": "2023-01-01T00:01:30Z"
}
```

**完了時のレスポンス:**
```json
{
  "task_id": "12345678-1234-5678-9012-123456789012",
  "status": "completed",
  "progress": 1.0,
  "message": "予測処理が正常に完了しました",
  "result": {
    "forecast_timestamp": [
      "2023-01-01T03:00:00",
      "2023-01-01T04:00:00",
      "2023-01-01T05:00:00"
    ],
    "forecast_values": [10.9, 11.3, 11.7],
    "model_name": "autoets_only",
    "confidence_intervals": {
      "lower_95": [10.5, 10.8, 11.0],
      "upper_95": [11.3, 11.8, 12.4]
    },
    "metrics": {"mse": 0.15, "mae": 0.12}
  },
  "created_at": "2023-01-01T00:00:00Z",
  "updated_at": "2023-01-01T00:05:00Z"
}
```

### 3. 予測タスクのキャンセル

```bash
curl -X DELETE "http://localhost:8000/api/v1/prediction_cancel/12345678-1234-5678-9012-123456789012"
```

**レスポンス:**
```json
{
  "task_id": "12345678-1234-5678-9012-123456789012",
  "message": "タスクが正常にキャンセルされました",
  "cancelled": true
}
```

### 4. 全タスクの一覧取得

```bash
curl -X GET "http://localhost:8000/api/v1/prediction_tasks"
```

## 予測ステータス

| ステータス | 説明 |
|-----------|------|
| `pending` | タスクが開始されました（待機中） |
| `running` | 予測処理を実行中 |
| `completed` | 予測処理が正常に完了 |
| `failed` | 予測処理でエラーが発生 |
| `cancelled` | タスクがキャンセルされました |

## リクエストパラメータ

### ZeroShotPredictionRequest

| パラメータ | 型 | 必須 | 説明 |
|-----------|---|------|------|
| `timestamp` | `List[datetime]` | ✓ | 時系列データのタイムスタンプ |
| `values` | `List[float]` | ✓ | 時系列データの値 |
| `forecast_until` | `datetime` | ✓ | 予測終了時点 |
| `model_name` | `string` | - | 使用するモデル名（デフォルト: "chronos_default"）。詳細は「利用可能なモデル」を参照。 |
| `model_params` | `Dict[str, Any]` | - | モデル固有のパラメータ |

## データ準備のベストプラクティス

### 推奨データ量
- **最小**: 予測期間の4倍以上のデータポイント
- **推奨**: 予測期間の10倍以上のデータポイント
- **例**: 24時間予測には最低96時間分（4日分）、推奨240時間分（10日分）

### データ品質
- タイムスタンプは時間順（昇順）に並べてください
- NaN値や無限値を含めないでください
- 一定間隔のサンプリングが推奨されます
- 極端な外れ値は事前に処理してください

### タイムスタンプ形式
ISO 8601形式を使用してください：
```
2023-01-01T00:00:00    # UTC
2023-01-01T00:00:00Z   # UTC明示
2023-01-01T09:00:00+09:00  # JST
```

## エラーハンドリング

### 一般的なエラー

| HTTPステータス | エラー内容 | 対処法 |
|---------------|-----------|--------|
| 400 | リクエストパラメータエラー | パラメータの形式や値を確認 |
| 404 | タスクが見つからない | 正しいtask_idを使用 |
| 500 | サーバー内部エラー | ログを確認、サポートに連絡 |

### 予測処理のエラー

予測処理中にエラーが発生した場合、タスクのステータスが`failed`になり、`error`フィールドにエラー詳細が記録されます：

```json
{
  "task_id": "12345678-1234-5678-9012-123456789012",
  "status": "failed",
  "error": "データポイントが不十分です（5点）。AutoGluonを使用した時系列予測には最低5つのデータポイントが必要です。",
  "message": "予測処理に失敗しました"
}
```

## パフォーマンスガイド

### 処理時間の目安

| モデル | 小規模データ（<100点） | 中規模データ（100-1000点） | 大規模データ（>1000点） |
|-------|-------------------|------------------------|----------------------|
| chronos_bolt | <10秒 | <30秒 | <1分 |
| seasonal_naive_only | <30秒 | <1分 | <2分 |
| autoets_only | <1分 | <3分 | <5分 |
| npts_only | <2分 | <5分 | <10分 |
| recursive_tabular_only | <3分 | <6分 | <10分 |
| fast_statistical | <2分 | <5分 | <8分 |
| balanced_ml | <5分 | <10分 | <15分 |
| deep_learning | <10分 | <20分 | <30分 |

### 並列処理
APIサーバーは複数の予測タスクを並列で処理できます（最大4並列）。
