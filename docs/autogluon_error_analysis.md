# zcrc-chronos API データ準備ガイドライン

## データ送信時の注意点

zcrc-chronos APIで高精度な予測を得るためのデータ準備ベストプラクティスを以下にまとめます。単一モデル専用設計により、各モデルの特性に応じた最適化が可能です。

### データ量と長さに関する要件

1. **最小データポイント数**: 少なくとも予測期間の4倍+100のデータポイントを提供してください
   - 例: 24時間の予測には、少なくとも (24 × 4) + 100 = 196 データポイントが必要
   - 理想的には予測期間の10倍以上のデータ量を推奨

2. **適切な予測期間の選択**:
   - 予測期間はデータ長の1/10以下に設定することを推奨
   - データが不足している場合、予測精度が低下するため注意

3. **サンプリング頻度の一貫性**:
   - 時系列データのサンプリング間隔が一定であることを確認
   - 不規則な間隔のデータは事前に補間することを推奨

### データ品質の確保

1. **欠損値の処理**:
   - NaN値や無限値（Inf）を含まないようにしてください
   - 欠損値がある場合は、送信前に適切な補間処理を実施

2. **異常値の対処**:
   - 外れ値や異常値が含まれている場合、モデル学習に悪影響
   - 極端な値は適切に処理するか、説明可能な範囲内に収める

3. **データ正規化**:
   - 可能であれば、データの範囲を統一するための正規化を検討
   - 特に大きな値の変動がある場合に効果的

### データフォーマットと構成

1. **タイムスタンプ形式**:
   - ISO形式の日時を使用（YYYY-MM-DDThh:mm:ss）
   - タイムゾーン情報が一貫していることを確認

2. **データの順序**:
   - 時系列データは古い順（時間的に昇順）に並べること
   - タイムスタンプと値の配列長が一致していることを確認

### エラー防止のためのヒント

1. **段階的なアプローチ**:
   - まず小さな予測期間でテスト
   - 正常に動作した後、徐々に予測期間を拡大

2. **季節性の考慮**:
   - データに明確な周期性がある場合、少なくとも2〜3周期分のデータを提供
   - 例: 日次の季節性には2〜3週間分、週次の季節性には2〜3ヶ月分

3. **データ前処理の共有**:
   - 特殊な前処理を行った場合はその情報も送信
   - モデルのパフォーマンスを向上させるヒントとなる
